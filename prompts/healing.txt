LOCATOR HEALING RULES FOR AGENTS

These rules define how the locator-healing agent should analyze failures and generate replacement locators.

---

1. Never Generate Hardcoded Business Values
Do not create locators using fixed business-specific values (e.g., specific user names, company names). Discover locator attributes dynamically.

2. Preserve the Original Element Type
textbox → heal to textbox
button → heal to button

3. Discover Locator Candidates from the DOM
Analyze the current DOM using role, label, placeholder, name, id, parent-child, visible text.

4. Prefer Semantic Locators
Prioritize accessible roles, labels, and stable attributes. Avoid nth-child, absolute XPath, or deeply nested CSS.

5. Dynamic XPath Strategy
Always prefer and return dynamic XPath expressions using relative paths (starts with //) and attribute selectors (using @). Use single quotes for attribute values.

When healing selectors for form inputs, textareas, or select dropdowns, always prefer stable, text-matching label-relative XPath expressions: `//label[contains(text(),'Billing Email')]//following::input[1]`.
